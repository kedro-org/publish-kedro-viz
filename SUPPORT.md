# Support

## Getting Help

This is an open-source project maintained by the Kedro community. We provide support through the following channels:

### Bug Reports & Feature Requests
- **GitHub Issues**: [Report bugs or request features](https://github.com/kedro-org/publish-kedro-viz/issues)
- **Response Time**: We aim to respond within 48-72 hours

### Community Support
- **GitHub Discussions**: [Ask questions and discuss](https://github.com/kedro-org/kedro/discussions)
- **Kedro Slack**: [Join the community](https://slack.kedro.org/)

### Direct Contact
- **Email**: kedro-framework@mckinsey.com
- **Use for**: Security vulnerabilities

## Support Policy

### What We Provide
- Bug fixes for supported versions
- Documentation improvements  
- Community-driven feature development
- Security vulnerability patches

### What We Don't Provide
- Guaranteed response times (best effort)
- Custom development services
- Paid support contracts
- Refunds (free product)

### Supported Versions
We support the latest major version. Security fixes may be backported to previous versions on a case-by-case basis.

## Marketplace and Legal Policies

### Takedown and Termination
This action is subject to GitHub's Marketplace policies and Terms of Service. In the event of:

- **Policy Violations**: If this action violates GitHub Marketplace policies, GitHub may remove it from the marketplace
- **Legal Issues**: DMCA takedowns or other legal challenges may result in temporary or permanent removal
- **Maintenance Issues**: Severe security vulnerabilities may require immediate takedown

### Internal Response Procedures
If GitHub flags or removes this action:

1. **Immediate Response** (within 24 hours):
   - Assess the violation or issue reported
   - Contact kedro-framework@mckinsey.com immediately
   - Document the circumstances and GitHub's reasoning

2. **Remediation** (within 48-72 hours):
   - Address technical issues, security vulnerabilities, or policy violations
   - Implement necessary fixes in the codebase
   - Prepare documentation of remediation efforts

3. **User Communication**:
   - Update repository README with status information
   - Create GitHub issue explaining the situation to users
   - Provide alternative solutions if action remains unavailable

- **Legal/Compliance**: kedro-framework@mckinsey.com
